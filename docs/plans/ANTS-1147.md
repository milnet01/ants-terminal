# ANTS-1147 — themedstylesheet extraction implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans
> to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Extract `MainWindow::applyTheme`'s ~228 LoC of QSS string-building into pure functions in `src/themedstylesheet.{cpp,h}` per `docs/specs/ANTS-1147.md`. Land the `updateStatusBar` cache-and-compare optimization in the same commit per spec § Behavioural-parity. No behavioural change otherwise.

**Architecture:** Six pure functions in namespace `themedstylesheet`, each `(Theme, …args) → QString`. No state, no class. MainWindow's `applyTheme` keeps the side effects (`qApp->setStyleSheet`, `setBackgroundFill`, dialog re-polish loop) and delegates string-building to the helpers. Three pre-1147 chip-QSS sites (`applyTheme:3118`, `updateStatusBar:4180`, `refreshRepoVisibility:4911`) collapse onto one `buildChipStylesheet` helper with an int margin parameter.

**Tech Stack:** Qt 6, C++20, CMake. No new dependencies.

**Spec:** `docs/specs/ANTS-1147.md` is the source of truth for behaviour, the eight INVs, the cache-and-compare optimization shape, and the `%23`-pre-encoding migration contract. Consult the spec for any "what" question.

**Buildable-state contract:** as with 1146, the carve-out can't sit half-extracted across a commit boundary. Tasks 2 + 3 land in one commit (skeleton + body migration + call-site swaps + optimization).

## Risk register

- **R1 — `%23` SVG pre-encoding migration.** If the comment doesn't migrate or the splice is dropped, the tab-close glyph hover stops painting (Qt's CSS parser truncates the data-URI at `#`). Mitigation: spec INV-2 asserts `data:image/svg+xml;utf8` is in the new TU; smoke step 2.11 visually verifies the glyph (offscreen Qt won't show it but a real window would).
- **R2 — Cache invalidation on theme switch.** If `applyTheme` doesn't clear `m_lastBranchChipValid`, switching themes leaves the chip with the previous theme's colours until the user changes branches. Step 2.6 explicitly invalidates.
- **R3 — Chip helper margin parameter signed/unsigned.** Spec uses `int leftMarginPx`. Negative values are nonsensical but not blocked; document via comment that callers pass the literal pixel count from the original QSS.
- **R4 — INV-7 LoC floor (≥200).** If extraction is partial, INV-7 fails. Cold-eyes review tightened from ≥150 to ≥200 to match the spec arithmetic.
