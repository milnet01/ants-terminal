**Goal:** Extract `ClaudeStatusBarController` from `mainwindow.cpp` into `src/claudestatuswidgets.{cpp,h}` per `docs/specs/ANTS-1146.md` — shrink `mainwindow.cpp` by ≥ 480 LoC, no behavioural change.

**Architecture:** New top-level `class ClaudeStatusBarController : public QObject` owning four Claude status-bar widgets (label, context bar, review button, error label) + bg-tasks tracker/button + state booleans + permission button-group factory + tab-indicator provider. Setup choreography in `MainWindow` ctor: construct → wire providers → `attach()` → `applyTheme()` → connect six signals.

**Tech Stack:** Qt 6, C++20, CMake. No new dependencies. New TU links into the main `ants-terminal` target alongside existing source files.

**Spec:** `docs/specs/ANTS-1146.md` is the source of truth for behaviour, public API, INVs, and the 12-row external-touch-site rewrite map. This plan describes execution order and intermediate build states; consult the spec for any "what" question.

**Buildable-state contract:** every task produces a buildable tree (compiles + existing ctest green). The carve-out can't be half-done across a commit boundary — Tasks 2 and 3 between them migrate the body in one atomic step, but Task 2's scaffold lets Task 3 be a code-move with no API surprises.

```cpp
void ClaudeStatusBarController::setError(const QString &text,
                                         const QString &tooltip,
                                         int autoHideMs) {
    if (!m_errorLabel) return;
    m_errorLabel->setText(text);
    m_errorLabel->setToolTip(tooltip);
    m_errorLabel->show();
    QTimer::singleShot(autoHideMs, m_errorLabel, &QWidget::hide);
}

void ClaudeStatusBarController::clearError() {
    if (m_errorLabel) m_errorLabel->hide();
}

void ClaudeStatusBarController::resetForTabSwitch() {
    m_lastState = ClaudeState::NotRunning;
    m_lastDetail.clear();
    m_planMode = false;
    m_auditing = false;
    if (m_reviewBtn)    m_reviewBtn->hide();
    if (m_contextBar)   m_contextBar->hide();
    if (m_bgTasksBtn)   m_bgTasksBtn->hide();
    if (m_bgTasks)      m_bgTasks->setTranscriptPath(QString());
    apply();
}
```

## Risk register

- **R1 — `permissionRequested` lambda's `currentTerminal` provider check.** The lambda needs `currentTerminal()` to find the active terminal for routing the awaiting-input flag. If the provider isn't set when the lambda fires (window startup race), the existing fallback "use the active tab" logic must still hold. Mitigation: verify in Task 4.7 step 3 (permission prompt smoke).
- **R2 — Theme freshness on contextUpdated.** The `contextUpdated` lambda reads `m_currentThemeName`. If `applyTheme` hasn't been called before the first context tick, the lambda has an empty theme name → `Themes::byName("")` → fallback. Mitigation: `MainWindow` ctor calls `applyTheme(m_currentTheme)` *before* any `m_integration` event can fire (no terminal exists yet). Already in the choreography.
- **R3 — Q_OBJECT moc regen.** Adding a new `Q_OBJECT` class requires CMake's AUTOMOC to re-run. If the build is incremental with stale moc state, missing-vtable link errors appear. Mitigation: `cmake --build build --clean-first` if any link error mentions vtable / staticMetaObject.
- **R4 — Existing test `claude_state_dot_palette`'s `Glyph::Auditing` count.** I might be wrong about the count being exactly 2 across the new file. If the tab-bar provider lambda doesn't reference `Glyph::Auditing` (it routes via the broader switch and only the `if (s.auditing && ...)` branch sets it), the actual count post-migration could differ. Mitigation: in Task 4.5 actually count occurrences and adjust the INV threshold rather than assuming.
