### Fix 3 — M4 — Kitty image cache evicts oldest entry, not all

- **File:** `src/terminalgrid.cpp:1607`; add insertion-order tracking in `src/terminalgrid.h`.
- **Change:**
  - Add `std::deque<int> m_kittyImageOrder;` to `TerminalGrid` private section.
  - Replace the `.clear()` branch with a `pop_front` loop that evicts entries until size < `MAX_KITTY_CACHE`, and erases the matching keys from `m_kittyImages`.
  - On erase-by-id (when an image with the same id is re-stored), remove the old entry from the deque first.
- **Verify:** build; `printf` a sequence of 201 distinct Kitty images with ids 1..201 and inspect that id=1 is the only one dropped (not ids 1..199).
- **Blast radius:** one function in `terminalgrid.cpp`. No public API change.
- **Rollback:** revert the commit.

### Fix 4 — M5 — `Config` getters bound numeric values

- **File:** `src/config.cpp` (getters only).
- **Change:** for each numeric getter that has a clamped setter, apply the same `qBound` in the getter. Scope: `fontSize`, `scrollbackLines`, `opacity`, `backgroundAlpha`, `aiContextLines`, `terminalPadding`. String getters unchanged.
- **Verify:** hand-edit `~/.config/ants-terminal/config.json` with `"font_size": 9999`, launch, confirm the UI doesn't misbehave and the rendered font is clamped to 48.
- **Blast radius:** ~6 getters. Read-only, no saves.
- **Rollback:** revert.

### Fix 5 — M6 — tighten Claude process-name match

- **File:** `src/claudeintegration.cpp:75-86`
- **Change:** split `cmdline` on `\0` (we already do a replace, keep argv[] pre-split), inspect argv[0] basename; match exact `{claude, claude-code}`. Fallback: if argv[0] is `node` / `deno` / `bun`, check argv[1..] for a path whose basename is `claude` or `claude-code`.
- **Verify:** run `grep claude somefile` in a child shell — Ants must NOT flip to Claude-running state. Run real `claude` — must flip.
- **Blast radius:** detection-only; false negatives would just show "not running" (graceful degradation).
- **Rollback:** revert.

### Fix 6 — M1, M2 — tab-rename / tab-color lambdas use `QWidget*`, not `tabIndex`

- **File:** `src/mainwindow.cpp:2276-2320`, plus `closeTab` for stale `m_tabColors` cleanup.
- **Change:**
  - In both lambdas: `QWidget *w = m_tabWidget->widget(tabIndex);` at menu-show time; capture `w` (not `tabIndex`). At click time: `int idx = m_tabWidget->indexOf(w); if (idx < 0) return;`, then use `idx`.
  - In `closeTab`: also remove the closed tab's entry from `m_tabColors` keyed by the new widget-pointer scheme (or switch `m_tabColors` hash key to `QWidget*` for symmetry).
- **Verify:** open three tabs, right-click tab 2 → Rename, close tab 1 before pressing Enter; confirm tab 2's label changed, not tab 1's (and no crash).
- **Blast radius:** menu handlers only.
- **Rollback:** revert.

### Fix 7 — H2 — save/restore SGR attrs on DECSET 1049/47/1047  ⚠ confirm

- **File:** `src/terminalgrid.cpp:320-348` (enter), `:373-384` (exit); header `src/terminalgrid.h` for new member.
- **Change:**
  - Add `CellAttrs m_altSavedAttrs;`, `bool m_altSavedOriginMode = false;`, `bool m_altSavedAutoWrap = true;` to `TerminalGrid`.
  - On enter: `m_altSavedAttrs = m_currentAttrs;` + save origin/wrap.
  - On exit: restore the three.
- **Verify:** `printf '\e[1;31mRED\e[?1049h\e[32mALT\e[?1049lTAIL\n'` — `TAIL` must render bold-red, not default.
- **Blast radius:** alt-screen transition only; touches colour/state handling used by `vim`, `less`, `htop`. Low risk of regression but many users hit this path every session.
- **Confirm before executing:** This changes user-visible behaviour for every TUI app that uses 1049. I want to verify against xterm's observable behaviour on the same sequence to make sure our restore-set matches, not just "looks right on one example."
- **Rollback:** revert.

### Fix 8 — H1 — OSC 8 hyperlink scheme allowlist  ⚠ confirm

- **Files:** `src/terminalwidget.cpp:2510` (detectUrls site) — or better, at the OSC 8 storage point in `src/terminalgrid.cpp:559-604` so bad URIs are dropped once, not validated twice.
- **Change:** before storing `m_hyperlinkUri`, parse its scheme (`uri.substr(0, uri.find(':'))`, lowercase) and reject anything outside `{http, https, file, mailto, ftp}`. Close-hyperlink path is unchanged.
- **Verify:** `printf '\e]8;;javascript:alert(1)\e\\TEST\e]8;;\e\\\n'` then Ctrl+click — nothing opens, no error dialog, `TEST` still underlined/hoverable if the terminal style does that, but no URL action.
- **Blast radius:** only OSC 8 explicit hyperlinks. Regex-detected URLs already go through their own scheme regex.
- **Confirm before executing:** I'd like to decide on the allowlist with you — the Egmont gist mentions `http`/`https`/`ftp`/`file`/`mailto` as common; STANDARDS.md lists `http/https/file/mailto`. Should `ftp` be in or out? I recommend **in** to match the detection regex at `terminalwidget.cpp:2487`, for consistency.
- **Rollback:** revert.

### Fix 9 — M3 — reflow preserves combining characters  ⚠ confirm

- **File:** `src/terminalgrid.cpp:1140-1241`.
- **Change:**
  - For each source `TermLine` joined into a logical line, record `(position_in_logical, cp)` for every combining codepoint (position = current `logical` length + per-line column).
  - After re-wrapping the logical line into width-`cols` chunks, redistribute the collected combining records into the new `TermLine::combining` maps using modulo / division math on the recorded position.
- **Verify:** paste `café résumé naïve` into a narrow terminal, shrink the window to force wrap, expand. Accents must survive.
- **Blast radius:** reflow is a hot spot; prior audits have both added and removed bugs here. A bad regression affects every resize.
- **Confirm before executing:** Given reflow fragility, I'd like to land this one **last** so a regression is easy to bisect. Also considering: skip this fix and flag it for a dedicated reflow rewrite instead.
- **Rollback:** revert.

### Fix 10 — H3 — decide on hook transport  ⚠ confirm

- **File:** `src/claudeintegration.cpp:813-862`.
- **Change:** **investigate first, then act.** My current read is that `ensureHooksConfigured()` writes hook commands that pipe to a *file* while the hook server listens on a *socket*. Nothing reads the file; nothing writes to the socket. Either the file path is stale (to be deleted) or the socket path is stale (to be rewired). Until I know which, I don't want to patch blindly.
  - Option A: delete `ensureHooksConfigured()` entirely and rely on whatever writes to the socket today. Verify by disconnecting the file-write and confirming events still reach the UI.
  - Option B: rewrite the hook command to `nc -U /tmp/ants-claude-hooks-$PID` (or a small helper) so events land on the socket the server already listens on.
- **Confirm before executing:** I want to read the actual invocation path (how Claude Code picks up these hooks today) with you in the loop before editing.
- **Rollback:** revert.

### Fix 11 — M7 — MCP JSON-RPC envelope  ⚠ confirm

- **File:** `src/claudeintegration.cpp:495-610`.
- **Change:** wrap every response as `{"jsonrpc":"2.0","id":<echoed>,"result":{...}}`, echo the request's `id`, and emit JSON-RPC errors (`{"jsonrpc":"2.0","id":N,"error":{"code":-32601,"message":"Method not found"}}`) for unknown methods. Also add a minimal `initialize` method so compliant clients can handshake.
- **Verify:** use the MCP inspector CLI (`mcp-inspector` or equivalent) to run `tools/list` against our socket and confirm a well-formed response.
- **Confirm before executing:** Only worth doing if we actually want an external MCP client to talk to Ants. If the existing integration is purely internal (same-process Claude Code hooks), this is scope creep. Decide together.
