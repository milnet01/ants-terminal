# ANTS-1014 ‚Äî Clipboard-write redaction helper

- **Status:** Implementing (pre-0.7.63)
- **Source:** indie-review-2026-04-27 (carry from 7th-audit memory)
- **Lanes:** TerminalWidget, MainWindow (Lua plugin glue), new
  `src/clipboardguard.{h,cpp}` module.

## Problem

Today the codebase has **15 raw `QApplication::clipboard()->setText(...)`
sites**, 13 in `terminalwidget.cpp` and 2 in `mainwindow.cpp`. There
is no central funnel ‚Äî each call site copies whatever bytes it has,
whatever the source. The 7th-audit memory flagged the OSC 52 callback
(`terminalwidget.cpp:97`) as the headline exfiltration vector: any
process running under the PTY can write arbitrary text to the user's
system clipboard without confirmation, without rate limiting, and
without size capping at the clipboard layer (the size cap inside the
OSC 52 parser layer caps the *parse*, not the clipboard write ‚Äî
inconsistent surfaces).

The other untrusted source is the Lua plugin `ants.clipboard.write`
capability (`mainwindow.cpp:511`). Plugins can be third-party, so the
trust shape matches OSC 52 in principle even if the threat model is
different.

The remaining 13 sites are all user-initiated UI actions (mouse
selection, context-menu "Copy Link" / "Copy Command", `copySelection`,
"copy last command output", URL quick-select fallback) ‚Äî these are
"trusted" in that the user just clicked something whose semantics
include a clipboard write.

## Fix

A single funnel: `clipboardguard::writeText(text, source, mode)` in
`src/clipboardguard.{h,cpp}`. All 15 call sites convert.

The helper splits into:

- **`clipboardguard::sanitize(text, source)`** ‚Äî *pure* free function
  in the header, no Qt dependency beyond `QString`. Returns a
  sanitized copy of `text` per the source's policy. Testable
  without spinning up `QApplication`.
- **`clipboardguard::writeText(text, source, mode)`** ‚Äî wrapper in
  the .cpp that calls `QApplication::clipboard()->setText(
  sanitize(text, source), mode)`. The .cpp depends on Qt6::Gui;
  the header does not.

### Source taxonomy

```cpp
enum class Source {
    Trusted,         // user UI action ‚Äî selection, context menu, etc.
    UntrustedPty,    // OSC 52 from the shell process
    UntrustedPlugin, // ants.clipboard.write from a Lua plugin
};
```

### Sanitization policy per source

| Source | NUL strip | Size cap | Notes |
|--------|-----------|----------|-------|
| `Trusted` | ‚úÖ | none | Defensive NUL strip catches malformed input; user actions can paste anything they want |
| `UntrustedPty` | ‚úÖ | **1 MiB** | OSC 52 streams from a shell process; cap defends against accidental DoS via `cat /dev/urandom \| base64` etc. Existing OSC 52 parser already has its own cap, so this is defense-in-depth. |
| `UntrustedPlugin` | ‚úÖ | **1 MiB** | Same threat-model shape as PTY; a malicious plugin shouldn't be able to spam the system clipboard with unbounded text. |

NUL stripping is universal because `QString` doesn't store NULs as
`\0` bytes natively (it stores QChar(0) which round-trips through
clipboard managers in unpredictable ways), and a NUL-bearing
clipboard payload is the kind of thing only attackers care about.

We deliberately do **NOT** apply pattern-based "secret detection"
(AWS keys, Bearer tokens, etc.) at this layer ‚Äî the user's terminal
is the user's tool, and "I want to copy this token to paste into
another window" is a legitimate primary use case. False-positive
redaction would be worse than no redaction.

### Out of scope

- Pattern-based secret detection (above).
- Rate limiting at the clipboard layer (OSC 52 parser already does
  this at the PTY-stream layer).
- Confirmation dialogs for OSC 52 writes (already a separate user
  preference at the parser layer, gated by `osc52_confirm_threshold`
  in config ‚Äî not duplicated here).
- Selection-mode (`QClipboard::Selection`) special handling ‚Äî same
  policy as the regular clipboard.

## Acceptance (INVs)

Tests live in `tests/features/clipboard_redaction/`. Source-grep +
behavioural drive of the pure `sanitize` helper.

- **INV-1** Header `src/clipboardguard.h` declares the
  `clipboardguard` namespace, the `Source` enum class with the three
  documented values (Trusted, UntrustedPty, UntrustedPlugin), and
  an inline (or free function) declaration for `sanitize(QString
  text, Source source) -> QString`. Asserted by source-grep.
- **INV-2** `clipboardguard::sanitize(text, Trusted)` strips embedded
  `QChar(0)` characters from the input. Behavioural drive.
- **INV-3** `clipboardguard::sanitize(text, UntrustedPty)` truncates
  text exceeding 1 MiB to 1 MiB + appends nothing (silent
  truncation; the OSC 52 parser layer already surfaces a status
  message for size events). Behavioural drive.
- **INV-4** `clipboardguard::sanitize(text, UntrustedPlugin)` applies
  the same 1 MiB cap as `UntrustedPty`. Behavioural drive.
- **INV-5** `clipboardguard::sanitize(text, Trusted)` does NOT
  truncate large inputs ‚Äî user pasting a 10 MiB file dump from
  context menu is a legitimate flow. Behavioural drive.
- **INV-6** No raw `QApplication::clipboard()->setText(` call survives
  in `src/terminalwidget.cpp` ‚Äî every site converts to
  `clipboardguard::writeText(...)`. Asserted by negative source-grep.
- **INV-7** No raw `QApplication::clipboard()->setText(` call survives
  in `src/mainwindow.cpp` ‚Äî both the Lua plugin glue (line 511) and
  the git-context hook (line 5996) convert. Asserted by negative
  source-grep.
- **INV-8** Source classification at each call site matches the
  taxonomy: the OSC 52 callback uses `Source::UntrustedPty`; the Lua
  plugin glue uses `Source::UntrustedPlugin`; everything else uses
  `Source::Trusted`. Asserted by source-grep on each line.
