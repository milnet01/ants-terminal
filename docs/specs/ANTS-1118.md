# ANTS-1118 — Smooth-scroll snapshot race during streaming

- **Status:** Implementing (target 0.7.65)
- **Source:** user-2026-04-30 (downgraded MEDIUM 2026-05-01) +
  indie-review-2026-05-01 (L2 root cause confirmed)
- **Lanes:** TerminalWidget, terminalgrid (snapshot path)

## Problem

When the user wheel-scrolls up while a Claude Code stream is
landing batches, the visible viewport briefly shows
overwritten text — but the underlying scrollback buffer is
intact (scrolling back to the same area paints correctly).
The 0.7.61 diagnostic-logging commit (`1abf768`,
`ANTS_DEBUG=scrollback`) traces `onVtBatch` but is gated on
`wasPinned = m_scrollOffset > 0` — and the racy batches are
exactly the ones where `m_scrollOffset` is still 0.

## Root cause (per indie-review L2 fold-in)

`TerminalWidget::wheelEvent` (terminalwidget.cpp:1828) sets
`m_smoothScrollTarget += lines` and starts the 16 ms
`m_smoothScrollTimer`. **`m_scrollOffset` stays at 0** until
the first `smoothScrollStep` tick commits a non-zero `intStep`.

`smoothScrollStep` (terminalwidget.cpp:2070):

```cpp
double step = m_smoothScrollTarget * 0.30;        // 0.9 for one notch
if (std::abs(step) < 0.5) step = ... 0.5 ...;     // 0.9 ≥ 0.5, no clamp
m_smoothScrollCurrent += step;                     // 0.9
m_smoothScrollTarget -= step;                      // 2.1

int intStep = static_cast<int>(m_smoothScrollCurrent);  // int(0.9) = 0
if (intStep != 0) { ... updateScrollBar(); ... }   // skipped
```

So the first 16–32 ms tick produces no offset transition and
no snapshot capture. During that window:

- `setScrollbackInsertPaused(false)` (still at default — the
  `onVtBatch` setter at line 1915 reads `m_scrollOffset > 0`
  which is false).
- Any `VtBatch` that arrives runs `m_grid->processAction(...)`
  — mutating screen rows the user is trying to scroll past.
- `m_syncOutputActive` is OFF (Claude Code / Ink doesn't emit
  DEC mode 2026), so `update()` runs at line 2034 — the user
  sees the mutated row.

Second tick: `intStep` finally becomes 1, `m_scrollOffset`
transitions to >0, `updateScrollBar()` calls
`captureScreenSnapshot()` — but the snapshot now copies the
**post-mutation** grid. The "stable" frozen view already
contains the overwrite.

User keeps scrolling up. Eventually they reach pure scrollback
(which IS correct because by then `setScrollbackInsertPaused`
is true). That matches the user report verbatim: *"When I
scroll back to the affected area, the text shows correctly."*

## Fix

Trigger snapshot capture + `setScrollbackInsertPaused(true)`
on **scroll intent** (positive `m_smoothScrollTarget` from
offset 0 with no existing snapshot) rather than on committed
`m_scrollOffset` transition.

Two surgical edits in `terminalwidget.cpp`:

1. **`wheelEvent` — after the target clamp + before the timer
   start.** When the user wheels up from a position where
   `m_scrollOffset == 0` and `m_frozenScreenRows.empty()`,
   capture the snapshot and pause scrollback insertion
   synchronously. Subsequent `onVtBatch` calls within the
   race window now correctly see `setScrollbackInsertPaused`
   already true (mid-batch the existing setter at line 1915
   re-affirms it; both sources agree).

2. **`smoothScrollStep` — timer-stop branch.** When the
   target settles back to 0 without ever committing a step
   (e.g. user wheels up then down before a tick fires
   `intStep != 0`), call `updateScrollBar()` once so the
   stranded `wantFrozen=false ∧ haveFrozen=true` snapshot is
   cleared. `updateScrollBar` is idempotent and the call is
   cheap.

`Shift+PageUp` / `Shift+Home` / `Shift+Up` already work
correctly because they set `m_scrollOffset` non-zero
synchronously — the existing `updateScrollBar()` call captures
the snapshot before any batch can race. No change needed there
beyond a code comment.

## Acceptance (INVs)

Tests live in `tests/features/scroll_snapshot_intent/`.
Source-grep + behavioural drive of the cleanup helper.

- **INV-1** `TerminalWidget::wheelEvent` body contains a
  `captureScreenSnapshot()` call gated on
  `m_scrollOffset == 0 && m_smoothScrollTarget > 0` (or
  equivalent intent-positive predicate) AND
  `m_frozenScreenRows.empty()`. Asserted by source-grep.
- **INV-2** The same body calls
  `m_grid->setScrollbackInsertPaused(true)` in the same
  branch. Asserted by source-grep.
- **INV-3** `TerminalWidget::smoothScrollStep` body, in the
  timer-stop branch (`std::abs(m_smoothScrollTarget) < 0.01`),
  calls `updateScrollBar()` (or equivalent snapshot-cleanup
  helper) before returning. Asserted by source-grep.
- **INV-4** Defensive: the existing `onVtBatch`-side
  `setScrollbackInsertPaused(m_scrollOffset > 0)` call at
  line 1915 is preserved (no regression of the existing
  pinned-pause invariant). Asserted by source-grep.
